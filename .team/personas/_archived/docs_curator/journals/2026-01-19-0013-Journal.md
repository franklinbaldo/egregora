**Observation:** During my routine audit of the `README.md`, I discovered that the "Preview your site" command was broken for users who installed Egregora via `git+https`. The documented command `uv tool run --with ".[mkdocs]" ...` relied on the presence of a local `pyproject.toml`, which end-users typically do not have. This led to a confusing experience where users could initialize a site but not preview it.

**Action:**
1.  Verified the failure by simulating a user environment (outside the repo root).
2.  Explored alternative commands and discovered that `uv tool run --from "git+https://github.com/franklinbaldo/egregora[mkdocs]" mkdocs serve ...` is the most robust solution. It ensures version consistency and correctly installs the necessary plugins without requiring a local source checkout.
3.  Updated `README.md` to use this new, reliable command.
4.  Verified that the `egregora demo` command works (and even suggests the long-form command, which I've now simplified in the docs).
