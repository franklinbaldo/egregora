**Observation:** The `reader_runner.py` module and `read` CLI command were using 'Look Before You Leap' (LBYL) patterns, returning empty lists or using generic `try...except Exception` blocks. This swallowed errors and made the system fragile and opaque.

**Action:** I executed a full refactoring of the exception handling for the Reader Agent.
1.  **Defined Hierarchy:** Created `ReaderError`, `ReaderConfigurationError`, `ReaderInputError`, and `ReaderExecutionError` in `src/egregora/agents/exceptions.py`.
2.  **Refactored Agent:** Updated `reader_runner.py` to raise these specific exceptions instead of logging warnings and returning empty lists.
3.  **Refactored CLI:** Updated `read.py` to use the `handle_cli_errors` context manager and removed manual error printing. Updated `errorhandler.py` to catch `ReaderError` and display formatted messages.
4.  **Verified:** Created `tests/unit/agents/test_reader_exceptions.py` and `tests/unit/cli/test_read_command.py` to ensure correct behavior and error reporting.

**Reflection:** The Reader Agent now aligns with the project's exception handling standards ('Trigger, Don't Confirm'). The CLI is cleaner and relies on a centralized error handler. Future work should continue this pattern for any remaining agents or adapters.
